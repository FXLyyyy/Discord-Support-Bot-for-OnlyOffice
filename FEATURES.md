# Discord Support Bot для OnlyOffice — реализованный функционал

Самостоятельный Discord-бот для поддержки пользователей: создание и ведение тикетов,
приватные каналы, транскрипты, интеграция с ONLYOFFICE DocSpace, внутренние заметки
персонала, статистика и автоматизация.

## Технологии

- Node.js + TypeScript
- discord.js v14 (slash-команды, кнопки, модальные окна, приватные треды)
- Supabase (PostgreSQL) — хранение тикетов, заметок, рейтингов, конфигурации
- ONLYOFFICE DocSpace (REST API) — постоянное хранилище транскриптов
- pdfkit — генерация PDF-транскриптов

## Создание тикетов

- Панель с кнопкой «Open a Ticket» (команда `/ticket-panel`), приветственный текст с инструкцией
- Создание тикета через модальное окно: тема и описание проблемы
- Для каждого тикета создаётся отдельный приватный канал, доступный только автору и персоналу
- Ограничение: один активный тикет на пользователя; при попытке открыть второй бот направляет в существующий
- В приветственном сообщении тикета прописаны правила (автозакрытие по неактивности, возможность переоткрытия, запрет на передачу паролей)
- Возможность прикреплять файлы и скриншоты прямо в канал тикета

## Жизненный цикл тикета

- Статусы: открыт -> взят в работу -> закрыт
- Взятие тикета в работу (`Claim`) с публичным уведомлением в канале
- Назначение тикета на конкретного агента руководителем (`/assign`)
- Закрытие тикета сотрудником с обязательным указанием решения (резолюции)
- Закрытие тикета самим пользователем (`/solved`)
- Закрытие с указанием внутренней причины (видна только персоналу)
- Автоматическое закрытие при неактивности: предупреждение через 24 часа, закрытие через 48 часов

## Переоткрытие тикетов

- Переоткрытие кнопкой прямо в закрытом канале — с полным сохранением переписки
- Переоткрытие по номеру тикета с панели (для пользователя — только своих тикетов)
- Переоткрытие персоналом по номеру (`/reopen`)
- При переоткрытии тикет автоматически назначается на прежнего агента, агент получает персональное уведомление
- Закрытые каналы не удаляются сразу, а архивируются (доступны только для чтения) и переносятся в отдельную категорию
- Автоматическая очистка архивных каналов старше 30 дней (с учётом лимитов Discord на количество каналов)

## Транскрипты и хранение в DocSpace

- При закрытии для каждого тикета создаётся отдельная папка в DocSpace
- В папку загружается PDF-транскрипт переписки (шапка, метаданные, решение, переписка)
- В ту же папку переносятся все файлы, вложенные пользователем
- Ссылка на папку с транскриптом сохраняется и доступна персоналу
- Полная версия транскрипта (с внутренними заметками) — для персонала; версия для клиента — без внутренней информации
- Транскрипт также отправляется пользователю в личные сообщения при закрытии тикета
- Параллельно ведётся HTML-транскрипт в канале логов

## Внутренние заметки персонала

- Команда `/internal` — добавление внутренней заметки, скрытой от клиента
- Заметки публикуются в приватный тред тикета, видимый только персоналу
- Заметки попадают в транскрипт персонала (с отдельным оформлением), но не в клиентскую версию
- Индикация наличия заметок: количество заметок отображается в `/ticket-info`

## Карточка вернувшегося пользователя

- При открытии нового тикета персонал получает сводку по прошлым обращениям пользователя
- Сводка содержит список прошлых тикетов с темами и ссылками на их транскрипты
- Для каждого прошлого тикета указывается количество внутренних заметок, тексты заметок выводятся прямо в сводке
- Сводка публикуется в приватном треде и сопровождается уведомлением персонала

## Оценка качества поддержки

- После закрытия тикета пользователю в личные сообщения приходит запрос оценки (1–5 звёзд)
- Оценки сохраняются в отдельную таблицу базы данных для последующего анализа

## Команды

- `/ticket-panel` — размещение панели создания тикетов (администратор)
- `/config` — настройка сервера: роли поддержки, канал логов, категория тикетов (администратор)
- `/claim` — взять тикет в работу (персонал)
- `/assign` — назначить тикет на агента (персонал)
- `/close` — закрыть тикет с резолюцией и внутренней причиной (персонал)
- `/solved` — закрыть тикет (пользователь или персонал)
- `/reopen` — переоткрыть тикет по номеру (персонал)
- `/add-user`, `/remove-user` — управление доступом к каналу тикета (персонал)
- `/internal` — внутренняя заметка (персонал)
- `/ticket-info` — информация о тикете (персонал)
- `/stats` — статистика поддержки

## Статистика

- Общее количество тикетов
- Количество открытых тикетов
- Количество закрытых за текущий месяц
- Среднее время закрытия
- Среднее время первого ответа
- Средняя оценка и количество оценок
- Топ-3 агента по числу обработанных тикетов

## Логирование и надёжность

- Все события и ошибки записываются в файлы логов (logs/bot-YYYY-MM-DD.log)
- Перехват необработанных исключений для последующего анализа
- Устойчивость к недоступности внешних сервисов: при отсутствии DocSpace бот не падает, а прикладывает транскрипт файлом
- Корректная работа без права Embed Links: ключевая информация дублируется обычным текстом

## Конфигурация

- Настройка ролей поддержки
- Настройка канала логов
- Настройка категории, в которой создаются каналы тикетов
- Конфигурация хранится отдельно для каждого сервера
